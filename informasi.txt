================================================================================
  BAGIAN 3: HAL YANG PERLU DIPERBAIKI / BUG POTENSIAL
================================================================================

1. ⚠️  METADATA LAYOUT MASIH DEFAULT
    File: app/layout.tsx (baris 15-18)
    Saat ini:
      title: "Create Next App"
      description: "Generated by create next app"
    Seharusnya diganti ke:
      title: "AI Code Generation — NLP Project"
      description: "Asisten pemrograman berbasis AI untuk autocomplete,
                     generate, refactor, unit test, dan dokumentasi kode."
    Impact: Kurang profesional jika dosen melihat tab browser.

2. ⚠️  LANG ATTRIBUTE LAYOUT
    File: app/layout.tsx (baris 27)
    Saat ini: lang="en"
    Karena aplikasi berbahasa Indonesia, pertimbangkan: lang="id"

3. ⚠️  README MASIH TEMPLATE DEFAULT
    File: README.md
    Masih berisi template bawaan create-next-app.
    Sebaiknya diganti dengan deskripsi project yang sebenarnya.

4. ⚠️  API KEY TEREXPOSE DI .env.local
    File: .env.local
    API Key Groq tertulis langsung. Pastikan file ini ada di .gitignore
    agar tidak ter-push ke repository publik.
    → Cek: .gitignore SUDAH include .env.local (aman), tapi pastikan
      repo GitHub-nya tidak berisi file ini.

5. ⚠️  TIDAK ADA ERROR HANDLING UNTUK GROQ API RESPONSE
    File: app/api/generate/route.ts (baris 41-43)
    Jika Groq API mengembalikan error (rate limit, invalid key, dll),
    kode tidak melakukan pengecekan res.ok sebelum parsing JSON.
    Bisa menyebabkan error tidak informatif di frontend.
    Saran: Tambahkan pengecekan:
      if (!res.ok) {
        return NextResponse.json(
          { error: "API Error: " + res.statusText },
          { status: res.status }
        );
      }

6. ⚠️  HANDLEGENERATE DEPENDENCY DI useEffect
    File: app/page.tsx (baris 160-172)
    handleGenerate tidak ada di dependency array useEffect.
    Ini bisa menyebabkan stale closure warning dari ESLint.
    Secara fungsional masih bekerja, tapi secara best practice
    sebaiknya di-wrap dengan useCallback.

7. ⚠️  TIDAK ADA AMBIENT BACKGROUND BLOBS
    Dari deskripsi disebutkan "Ambient animated background blobs
    (violet/indigo/fuchsia)" tapi di kode page.tsx tidak ada
    elemen background blob. Background hanya solid #0a0a0a.
    → Jika ingin menambahkan, perlu div absolute dengan gradient
      radial dan animasi.

8. ⚠️  TIDAK ADA LIBRARY clsx DAN tailwind-merge
    Dari deskripsi disebutkan menggunakan clsx dan tailwind-merge,
    tapi keduanya tidak ada di package.json dan tidak digunakan
    di kode manapun. Ini tidak masalah secara fungsional, tapi
    jangan disebut di presentasi jika tidak benar-benar digunakan.


================================================================================
  BAGIAN 4: SARAN DIFERENSIASI AGAR UNGGUL
================================================================================

Berikut saran fitur tambahan agar project ini menonjol dibanding
kelompok lain yang mungkin mengambil topik serupa:

1. 🌟 TAMBAHKAN HALAMAN "ABOUT / HOW IT WORKS"
   Buat halaman kedua yang menjelaskan arsitektur sistem:
   Frontend → API Route → Prompt Engineering → Groq API → Response
   Dengan diagram visual. Ini menunjukkan pemahaman mendalam.

2. 🌟 STREAMING OUTPUT (Server-Sent Events)
   Saat ini output muncul sekaligus setelah selesai.
   Jika menggunakan streaming, output akan muncul kata per kata
   seperti ChatGPT — sangat impressive saat demo.

3. 🌟 HISTORY/LOG GENERASI
   Simpan riwayat generate di localStorage sehingga user bisa
   melihat kembali hasil generate sebelumnya.

4. 🌟 MULTI-LANGUAGE SUPPORT
   Saat ini prompt backend fokus Python. Tambahkan dropdown
   untuk memilih bahasa (Python, JavaScript, Java, Go, dll.)
   agar lebih fleksibel. (Catatan: dari conversation history,
   sepertinya ini sudah pernah dipertimbangkan sebelumnya)

5. 🌟 DARK/LIGHT MODE TOGGLE
   Tambahkan toggle theme untuk menunjukkan fleksibilitas UI.

6. 🌟 EXPORT HASIL KE FILE
   Tambahkan tombol "Download as .py" atau "Download as .sql"
   agar user bisa langsung menggunakan kode yang digenerate.

7. 🌟 COMPARISON VIEW (Before/After)
   Khusus untuk tab Refactor dan Dokumentasi, tampilkan
   side-by-side view antara kode asli dan kode hasil AI.

8. 🌟 RESPONSE TIME INDICATOR
   Tampilkan berapa lama waktu yang dibutuhkan AI untuk
   men-generate kode (misal: "Generated in 1.2s").
   Ini menunjukkan kecepatan Groq yang jadi selling point.

9. 🌟 JUPYTER NOTEBOOK SEBAGAI PENDAMPING
   File Code_Generation.ipynb sudah ada. Pastikan notebook ini
   berisi demonstrasi yang complementary (bukan duplikasi)
   dari web app. Misalnya, tunjukkan perbandingan output dari
   berbagai temperature setting, atau evaluasi kualitas kode
   yang dihasilkan.

10. 🌟 FOOTER DENGAN CREDIT
    Tambahkan footer kecil: "Dibuat untuk Tugas NLP — [Nama] — 2026"
    dan link ke repository GitHub.

================================================================================
  BAGIAN 6: PROMPT ENGINEERING YANG DIGUNAKAN
================================================================================

Setiap use case memiliki system prompt spesifik:

1. AUTOCOMPLETE:
   "Kamu adalah asisten code generation. Lanjutkan kode Python
    berdasarkan komentar berikut. Hanya tulis kode, tanpa penjelasan."
   → Input fixed: # Fungsi untuk menghitung Fibonacci ke-n
     dengan memoization

2. GENERATE:
   "Kamu adalah asisten code generation. Tulis kode berdasarkan
    input berikut. Input bisa berupa kalimat biasa atau komentar kode.
    Hanya tulis kode, tanpa penjelasan."

3. NL → SQL:
   "Kamu adalah asisten SQL. Ubah kalimat berikut menjadi query SQL.
    Hanya tulis query SQL, tanpa penjelasan."

4. REFACTOR:
   "Kamu adalah asisten refactoring. Perbaiki dan rapikan kode berikut
    tanpa mengubah fungsinya. Hanya tulis kode hasil refactor,
    tanpa penjelasan."

5. UNIT TEST:
   "Kamu adalah asisten testing. Buatkan unit test Python menggunakan
    unittest untuk fungsi berikut. Hanya tulis kode unit test,
    tanpa penjelasan."

6. DOKUMENTASI:
   "Kamu adalah asisten dokumentasi. Tambahkan docstring dan komentar
    yang jelas pada kode berikut. Hanya tulis kode dengan dokumentasi,
    tanpa penjelasan tambahan."

Catatan penting:
- Semua prompt menggunakan instruksi "Hanya tulis kode, tanpa penjelasan"
  untuk memastikan output bersih (kode saja)
- Temperature 0.2 dipilih agar output deterministik dan akurat
- Output di-sanitasi dengan regex untuk menghapus markdown fences
